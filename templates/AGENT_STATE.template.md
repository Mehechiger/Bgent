# AGENT_STATE - 实时看板 (Dashboard)

<!--
  Bgent AGENT_STATE 模板
  =====================
  本文件是你项目的实时看板。将它放在项目根目录。
  AI Agent 会在每次会话启动时读取它、在结项仪式中更新它。

  五大板块顺序固定，不可调换：
    ① Deadlines  → 有硬截止日的事项
    ② Constraints → 精力/时间约束声明
    ③ Focus       → 当前冲刺周期的作战计划
    ④ Pending     → 全量待办（艾森豪威尔矩阵）
    ⑤ Achievements → 滚动 7 天成就记录

  使用说明：
  - 用你自己的任务替换 <示例> 内容
  - 保留板块标题和 emoji 前缀（脚本依赖它们解析）
  - 自定义类别标签（见 §4 说明）
-->

## ⏳ 待办事项 (Pending Actions — Eisenhower Matrix)

> 格式：`[类别] [Imp: X] [T•]` 前缀 + 任务详情
>
> **类别标签**（按你的项目自定义）：
> | 标签 | 用途 |
> |---|---|
> | `[Dev]` | 开发任务 |
> | `[Doc]` | 文档撰写 |
> | `[Admin]` | 行政事务 |
> | `[Deploy]` | 部署运维 |
> | `[Meeting]` | 会议沟通 |
>
> 你可以按需增删标签——框架不限制类别，只要求格式一致。
>
> **重要性 (Imp)**：1-5 分，5 为最高
> **时间开销 (T)**：T1 (< 1h) / T2 (1-4h) / T3 (> 4h)

### 🔴 第一象限：决战区 (A: Imp 4-5 + Deadline ≤ 2天)

- [ ] `[Imp: 5] [T2] [Dev]` 🔴 **功能 X 上线前关键 Bug 修复** `[D-1]`
- [ ] `[Imp: 4] [T1] [Admin]` **回复合作方紧急邮件** `[D-0]`

### 🟡 第二象限：储备区 (B: Imp 4-5 + Deadline > 2天)

- [ ] `[Imp: 5] [T3] [Dev]` **功能 Y 核心模块开发** `[⏰ 下周启动]`
- [ ] `[Imp: 4] [T2] [Doc]` **API 文档补全** `[Idle: 5d]`
- [ ] `[Imp: 4] [T2] [Admin]` **季度报告撰写** `[D-12]`

### 🔵 第三象限：流水线 (C: Imp ≤ 3 + Deadline)

- [ ] `[Imp: 3] [T1] [Admin]` **更新项目 Wiki** `[⚠️ Stale]`
- [ ] `[Imp: 2] [T2] [Doc]` **撰写内部技术分享 slides**

### ⚪️ 第四象限：背景池 (D: Imp ≤ 1 / Long-term)

- [ ] `[Imp: 1]` **调研新框架替代方案** — 长期技术储备
- [ ] `[Imp: 1]` **重构模块 Z 的测试覆盖** — 技术债

### 🗃️ 被挤出项 (Displaced Tasks)

> 存放因高优任务插入而被弹性置换的任务。
> 由 Agent 在排产提案中说明去向（退回资源池或延后）。

- `[Displaced MM-DD]` **任务名** — 原排周X HH-HH → **退回资源池**。理由：XXX
