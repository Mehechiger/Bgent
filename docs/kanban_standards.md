> **加载触发条件**：执行结项仪式、看板更新、排产调度或涉及精力约束时，Agent 必须首先读取本文件。
>
> 本文件是项目所有看板操作的"单一信息源"。

---

## 0. 看板板块规范 (Required Sections)

`AGENT_STATE.md` 必须包含以下五个板块，顺序固定：

| 板块 | 标题 | 用途 | 维护约束 |
|---|---|---|---|
| **①** | `⏳ 临近截止与倒计时` | 有明确时间节点的任务，按日期升序排列 | 必须标注 `[D-X]` 倒计时；已完成项保留 `[x]` 痕迹至多 5 天后清除 |
| **②** | `📵 当前约束声明` | 常规与临时精力约束（不含具体任务） | 排产前必须优先读取；临时约束过期后在结项仪式中清理 |
| **③** | `🎯 当前焦点` | 当前冲刺周期的作战计划，通常以周为单位 | **流式清理**：次日开启时，直接删除已成过去的整行日期 |
| **④** | `⏳ 待办事项` | 按艾森豪威尔矩阵四象限组织的全量待办 | 遵循纯加法原则；包含被挤出项缓冲区 `🗃️` |
| **⑤** | `✅ 近期成就` | 滚动记录过去 7 天的完成事项 | 超过 7 天的条目按 §3 规则清理或备份 |

## 1. 条目格式

**任务行标准格式**：
```
[类别] [Imp: X] [T1/T2/T3] 任务详情 [D-X] [Last: YYYY-MM-DD]
```

**类别标签**自定义（在 `AGENT_STATE.md` 模板中定义），框架不限制具体类别。

**重要性 (Imp)**：1-5 分，5 为最高。
**时间开销 (T)**：T1 (< 1h) / T2 (1-4h) / T3 (> 4h)。

## 2. 时效性标记规则

Agent 必须计算并更新以下标记：
- **[D-X] 倒计时**：针对有日期限定的任务
- **[Idle: Xd] 停滞追踪**：标记距离上次动作的天数
- **[⚠️ Stale] 预警**：非紧急任务停滞超过 7 天时自动标记

## 3. 条目生命周期与清理

- **进行中 (Active)**：使用 `- [ ]`。**禁止静默删除**任何未完成任务。
- **刚完成 (Just Completed)**：原地置为 `- [x]`。不要立即抹除，保留痕迹。同时在 Achievements 新增记录。
- **沉淀与清理**：
    - 已沉淀超 5 天的 `-[x]` 条目：彻底移除
    - **滚动成就 (Rolling 7 Days)**：超过 7 天的重大节点删前确认已备份；琐碎成就直接删除

## 4. 纯加法原则

> [!CAUTION]
> **看板更新纯加法原则 (Additive-Only)**：Agent 更新 `AGENT_STATE.md` 时，**严禁对原有条目进行删减、改写或信息简化**。
> 允许：在条目前 prepend 标签、更改完成状态、在条目后 append 补充信息。
> 如需重组结构（如调整象限），必须移动原文，不可重写原文。

| 时机 | 命令 |
|---|---|
| 会话初始化 | `scripts/daily_briefing.py --mode morning` |
| 结项仪式 | `scripts/daily_briefing.py --mode closing` |

## 6. 精力约束规则

具体约束值记录在 `AGENT_STATE.md § 📵 当前约束声明`。

**行为规则**：
1. **容量上限**：排产时不得超过当日声明的可用时长。完全 OFF 的日不排任务。
2. **提前量处理**：截止日任务落在约束日时，必须提前至更早的工作日完成。
3. **例外申请**：无法提前时必须向用户说明冲突，不得静默排产。

## 7. 滚动排产与调度协议

工作流采用**"资源池 + 执行表"双轨制**。

### 7.1 双轨结构
- **资源池** = `Pending Actions`：全量待办，按艾森豪威尔象限排列，"待调度"状态
- **执行表** = `Focus`：已确认时序的日程，"已锁定"状态

### 7.2 颗粒度规范

| 任务类型 | 精度要求 |
|---|---|
| **预约型**（会议、截止日） | 必须标注 `HH:MM` |
| **任务型**（开发、写作） | 精确到`日` |
| **背景型**（低 Imp） | 无时序要求，保留在资源池 |

### 7.3 排产仪式触发点

| 时机 | Agent 行为 |
|---|---|
| **会话开启** | 检查 Pending 🔴A/🟡B 类条目，提议将今日应处理项移入 Focus |
| **结项仪式** | 1. 流式清理：删除昨天及更早的日期行<br>2. 校验未完成任务不被静默删除 |
| **录入新 Deadline** | 同时在 Focus 占位 |

### 7.4 预警规则
- **[⚠️ Overloaded]**：当日预估总工作量 > 可用时长时发出过载预警
- **[⚠️ Unscheduled]**：Pending 中有 `[D-0]` 或 `[D-1]` 但未进入 Focus 时提醒

### 7.5 下周预览
- Focus 底部预留 `### 下周预览 (Next Week)` 区域
- 每周五结项仪式中，将预览条目固化为下周正式日程

### 7.6 弹性调度与置换规则

1. **白留白原则**：Focus 每日排产不超过可用容量的 **80%**
2. **末位淘汰**：高优任务插入时，扫描 Focus 中 Imp 最低且无硬 Deadline 的条目提议置换
3. **缓冲区存储**：被置换条目移入 `🗃️ 被挤出项`，标 `[Displaced]`，禁止直接删除
4. **原子确认**：排产变动必须先提交提案，经用户确认后执行

### 7.7 滑动窗口原则

Agent 只维护 **2 周视野**：
- **执行周 (W)**：日/时级精度
- **预案周 (W+1)**：大块任务和关键 Deadline
- **资源池 (W+2+)**：保留在 Pending Actions
- **窗口滑动**：每周五将 W+1 推进为 W

### 7.8 主动排产感知 (PRO)

> [!IMPORTANT]
> Agent 在任何导致 `AGENT_STATE.md` 变动的操作后，必须检查对排产的影响。
> 识别到新高优任务未排入 Focus 时，主动提议更新排产。
