# Protocol Details — 详细规则参考

本文档是 `meta_protocol.md` 的详细补充。Agent 在对规则执行方式不确定时应查阅此处。

## 记忆生命周期

### Daily Log（`.agent/memory/YYYY-MM-DD.md`）

- 初始化时仅加载**今天+昨天**
- 每 14 天检测一次超龄文件（`daily_briefing.py` 检测，`archive_memory.py` 执行归档），逾期顺延
- 归档 = 移动到 `.agent/memory/archive/YYYY/MM/`，不删除
- 归档后的记忆在必要时可按路径查阅

**写入格式**：
```markdown
## HH:MM — [对话主题简述]
- [做了什么，1句]
- 决策: [关键决策，如有]
- 新信息: [用户告知的新事实/偏好，如有]
- 下次注意: [后续对话需要知道的上下文，可为空]
```
省略无内容的字段。简单问答不触发写入。

### 写入工具约束

**允许**（有 diff，用户可审核）：
- `write_to_file`：新建 Daily Log
- `replace_file_content` / `multi_replace_file_content`：追加或编辑

**禁止**（无 diff 或受沙盒限制）：
- `run_command` + shell 写入（`echo >>`、`cat >>`、`sed -i` 等）
- `run_command` + 脚本直接写文件（`archive_memory.py` 除外）

**完整性保护**：
- 只许**追加**新条目，**禁止修改或删除**已有记忆内容
- 如需修改已有条目（纠错等），必须先说明原因并获用户批准

### 长期记忆（`.agent/MEMORY.md`）

- **写入 (双步确认)**：Agent 不能直接静默写入。必须先提炼表述，向用户发送预览确认后才能写入。
- **滚动审核**：上次审核超过 30 天后，脚本自动在晨间简报中提醒。Agent 审阅全部条目，对每条提议：
  - **删除**：已过时/不再相关
  - **提升**：稳定且有普适性 → 写入项目文档，从 MEMORY.md 删除
  - **保留**：仍在观察中
- 用户确认后执行。**平时不检测**。
- **审核完成后**：写入 `[审核 YYYY-MM-DD]` 标记

## 文档卫生

### 单一信息源 (Single Source of Truth)
- 任何特定主题的**完整详细内容**必须只存在于**一个**文档中
- 其他文档只引用或简要总结，不完整复制

## 看板追踪

### 两种用途
- **用途 A（会话开始）**：关注 **Focus** + **Achievements**，建立上下文
- **用途 B（任务结束）**：关注 **Pending Actions**，向用户建议下一步

### 回写内容
完成重要任务后更新 `AGENT_STATE.md`：
- 记录最新成就
- 刷新 `[D-X]` 倒计时和 `[Idle: Xd]` 标签
- 报告时效性状态

---

## 防遗漏协议

执行重构时的安全机制：
1. 修改前列出**关键功能点清单**
2. 执行修改
3. 修改后逐项确认清单中的每个功能点是否保留
